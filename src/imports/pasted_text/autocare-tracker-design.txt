Create a complete, modern, responsive web application UI/UX design for a platform named **AutoCare Tracker**.

AutoCare Tracker is a car maintenance and repair management system. It helps users organize all important information about their vehicles in one place: vehicle details, maintenance history, repairs, replaced parts, expenses, mileage, and future service reminders.

The design must be practical, easy to implement using React and Tailwind CSS, and suitable for a university bachelor project. Build the design using reusable Figma components, variants, Auto Layout, consistent spacing, and a clear design system.

Do not create only a few showcase screens. Create the complete application with every screen, reusable component, state, modal, form, and responsive version needed to implement the product.

---

# 1. PRODUCT GOALS

The application should allow users to:

1. Register and log in securely.
2. Add and manage one or multiple vehicles.
3. Record completed maintenance and repair work.
4. Track replaced parts and related expenses.
5. Set reminders based on date, vehicle mileage, or both.
6. View upcoming and overdue maintenance tasks.
7. Analyze vehicle-related expenses through charts and summary cards.
8. Use the application comfortably on desktop, tablet, and mobile devices.

The interface should feel reliable and organized. It should not feel overly complex or like professional enterprise fleet-management software. The primary target audience is an everyday car owner managing one or several personal vehicles.

---

# 2. DESIGN STYLE

Use a clean, modern automotive dashboard style.

The visual style should communicate:

* Reliability
* Safety
* Simplicity
* Good organization
* Professional vehicle maintenance
* Easy access to important information

Use a light theme as the main version.

Recommended visual direction:

* White and light-gray backgrounds
* Dark navy or charcoal text
* Blue as the main action color
* Green for completed or healthy statuses
* Orange for upcoming maintenance warnings
* Red for overdue maintenance or destructive actions
* Rounded cards with subtle shadows or borders
* Spacious layouts with clear information hierarchy
* Clear line icons related to vehicles, tools, calendars, money, charts, and settings

Avoid:

* Excessive gradients
* Neon colors
* Overly futuristic designs
* Heavy animations
* Crowded dashboards
* Decorative elements that make implementation difficult

Use a modern sans-serif font such as Inter.
